## 协商退触发逻辑优化 PRD 完成

### 核心决策
原「不可取消」场景中，协商退助手卡片由系统自动跟随下发 → 改为**关键词+上下文双条件触发**：

| 条件 | 说明 |
|------|------|
| 上下文 | 本会话已触发过「不可取消」告知 |
| 关键词 | 房客输入含 `退`/`商量`/`怎么办`/`能不能` |

两个条件同时满足才触发「平台协商退小助手」卡片，任一不满足则不触发。

### 工程实现要点
- session 级别记录「不可取消告知已触发」标记
- 字符串包含匹配（轻量，无需 NLU）
- 同一会话内协商退助手卡片只触发一次（去重）

### 付费取消场景不变
- 「取消二次确认」卡片内「尝试和房东协商」文字链仍保留，主动点击触发，不受本次规则影响
