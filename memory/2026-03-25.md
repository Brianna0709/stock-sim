## 技术问题记录

### OWA 发邮件
- OWA 是 Exchange 15.2.2562.27 Basic 版，不支持 service.svc JSON API（总报 OwaSerializationException）
- 必须用 agent-browser UI 自动化操作表单
- 收件人填写：多个用分号分隔（`a@meituan.com; b@meituan.com`）

### kmedit 超时问题（已修复 ✅）
- 根本原因：`browser.pages()` 在存在某些问题 tab（SSO 页等）时会挂死，导致所有 kmedit 命令 SIGTERM
- 修复文件：`/root/.meituan-local-tools/scripts/meituan-local-km-js/node_modules/@meituan/local-cdp-core/src/browser.ts`
- 修复方案：将 `await browser.pages()` 替换为 `browser.targets()` + 按需 `safeGetPage()`（带 3s 超时）
- 修复后验证：`kmedit apply` 正常在 ~15s 内返回，`ok: true`, save `responseStatus: 200`
- 注意：需同时修复 drawio-converter 里同版本的 browser.ts（路径不同但代码一样）
- DISPLAY=:1（不是 :0），xclip/xdotool 可作为 kmedit 的备用方案

### target 格式问题
- `kmedit apply` 的 target 必须有 `nodeId` 或 `fallback` 字段
- 直接写 `{"type":"heading","textContains":"xxx"}` 不行，需套 fallback：`{"fallback":{"type":"heading","textContains":"xxx"}}`
- heading 节点的 `nodeId` attr 在某些情况下存在（可用），在新创建节点中可能为 null
